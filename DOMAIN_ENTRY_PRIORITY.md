# Domain Entry Priority v1.4

## 1. Purpose

Neotro Protocol is a cross-industry state-reading protocol.  
However, business execution should begin with priority domains.

## 2. Priority Domains

### Battery

Priority reason:

- strong fit with BMS, ESS, EV telemetry, battery diagnostics, and degradation transition  
- strong message around no hardware replacement  
- strong public-data demonstration potential  
- lower regulatory burden than medical or autonomous final decision systems

Target lines:

- BMS Software  
- Battery Diagnostics  
- ESS Safety  
- Battery Analytics  
- EV / SDV Battery Platform

### Telco

Priority reason:

- strong fit with AI-RAN, AIOps, NOC, 6G, and autonomous network operations  
- strong retrospective log analysis potential  
- strong review-timing and transition-persistence framing  
- low need for hardware replacement

Target lines:

- AI-RAN  
- NOC / AIOps  
- Network Intelligence  
- 6G / Autonomous Network  
- AI Infra Operations

## 3. Secondary Domain

### OS / Work-state

Role:

- original Neotro state-order demonstration  
- human-AI work-state scenario  
- protocol alignment reference case

## 4. Restricted Domains

### Medical

Use only under:

- retrospective review  
- no diagnosis  
- clinician review only  
- no treatment recommendation  
- no automated clinical directive

### Autonomous

Use only under:

- offline log analysis  
- scenario review  
- no real-time actuation  
- no direct vehicle control  
- human or existing safety-stack review only
